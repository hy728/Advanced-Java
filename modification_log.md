# 项目开发 AI 辅助使用文档

## 一、团队协作概述

本项目为酒店预订系统的开发，采用前后端分离架构，后端基于 Spring Boot，前端使用 Angular 框架。团队成员包括组长黄莹、陆玟颖、庞雅丹、李悠扬、廖英伎、彭宁、李超婉、李乐琪等，在开发过程中合理利用 AI 工具辅助完成不同阶段的任务，提升开发效率和质量。黄莹作为组长，全面统筹项目进程，根据成员的技术专长与能力，为组员指定合理的分工，确保各环节任务分配科学高效。同时，负责收集整合全组成员提交的工作成果，仔细检查其中存在的问题并及时修正，保障项目整体的准确性与一致性。

## 二、各成员 AI 辅助工作说明
### 陆玟颖 - 环境配置与部署

- 使用 AI 工具：讯飞星火
- 应用场景：
  - 编写项目部署文档
  - 排查前端启动报错问题
- 目的：
  - 确保文档内容的专业性和准确性，使用 AI 将中文操作步骤准确翻译为英文技术文档
  - 通过 AI 分析解决 DOSKEY 命令找不到的环境配置问题，提供系统环境变量检查、Angular CLI 重新安装等解决方案

### 庞雅丹 - 前端开发与功能实现

- 使用 AI 工具：豆包
- 应用场景：
  - Angular 组件开发
  - 表单验证逻辑实现
  - API 调用代码生成
- 目的：
  - 快速生成组件模板代码，如房间列表展示、预订表单等
  - 自动补全表单验证逻辑，如日期格式检查、必填项验证等
  - 根据后端 API 文档自动生成 HTTP 请求代码，提高开发效率

### 李悠扬 - 多语言与货币功能开发

- 使用 AI 工具：豆包
- 应用场景：
  - 多线程语言翻译功能设计
  - 货币转换算法实现
  - 中英文文档翻译与术语统一
- 目的：
  - 利用 ChatGPT 设计多线程实现方案，确保不同语言欢迎消息的并行加载
  - 通过 AI 优化货币转换精度处理和汇率更新逻辑
  - 使用 DeepL 保证中英文技术文档术语的一致性和专业性

### 廖英伎 - 数据库设计与开发

- 使用 AI 工具：kimi
- 应用场景：
  - 数据库表结构设计
  - SQL 查询语句优化
  - 数据库索引规划
- 目的：
  - 通过 AI 生成规范化的数据库表结构，确保数据完整性和一致性
  - 优化复杂查询语句，提升数据库性能
  - 设计合理的索引方案，提高数据检索效率

### 彭宁 - 技术文档术语标准化

- 使用 AI 工具：豆包、专业术语词典
- 应用场景：
  - 中英文技术术语对照表整理
  - 文档术语一致性检查
  - 专业领域术语翻译
- 目的：
  - 建立统一的技术术语库，如将 "Database" 规范为 "数据库"，"Environment Variable" 规范为 "环境变量"
  - 确保文档中所有术语使用的一致性和准确性
  - 提供专业领域的准确术语翻译，避免歧义

### 李超婉 - AI 辅助文档整合

- 使用 AI 工具：豆包
- 应用场景：
  - 整合各成员工作内容
  - 编写使用文档
  - 优化文档结构与表述
- 目的：
  - 梳理各成员使用 AI 工具的工作内容，确保全面覆盖项目各环节
  - 撰写规范的使用文档，清晰说明 AI 工具的应用场景和目的
  - 优化文档结构，提高文档的可读性和专业性

### 李乐琪 - 开发过程可视化

- 使用 AI 工具：ChatGPT、截图工具
- 应用场景：
  - 关键开发步骤截图记录
  - 操作流程可视化文档生成
  - 问题排查过程记录
- 目的：
  - 记录项目开发过程中的关键步骤和操作界面，如 Git 克隆项目、Maven 构建过程等
  - 生成可视化的操作指南，便于团队成员参考和新成员学习
  - 记录问题排查过程，为后续技术支持提供参考
